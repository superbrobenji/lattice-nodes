# Error Codes – Seven-segment Display

The TM1637 4-digit display shows error codes that are generated by `ErrorCore::signalError()`.
Each code consists of **three hexadecimal digits** **T M S** returned by

```cpp
uint16_t code = makeErrorCode(T, M, sub);
```

| Pos. | Digit | Meaning |
|------|-------|---------|
| `T`  | Type  | 1 = Generic, 2 = Sensor, 3 = Comm, 4 = Memory, 5 = Hardware, 6 = Config |
| `M`  | Module| 1 = Core, 2 = Adapter, 3 = Mesh, 4 = EEPROM, 5 = Hardware-Abstraction |
| `S`  | Sub   | 0-F – module-specific qualifier |

Example `0x652` → T = 6 (Config) / M = 5 (Hardware) / S = 2.

## Registry of defined codes

| Code | Meaning |
|------|---------|
| 511  | Core – red LED failed to initialise |
| 512  | Core – EEPROM manager failed to initialise |
| 513  | Core – failed to create adapter |
| 514  | Core – adapter failed to initialise |
| 521  | Mesh – failed to read ESP32 MAC address |
| 532  | Mesh – peer list full |
| 533  | Mesh – ESP-NOW init failed |
| 534  | Mesh – invalid incoming ESP-NOW frame |
| 535  | Mesh – ESP-NOW send error |
| 536  | Mesh – multiple master nodes detected |
| 651  | Hardware – invalid GPIO pins for 7-segment display |
| 652  | Hardware – TM1637 didn’t acknowledge (bus error) |
| 651* | Hardware – LED given invalid pin number (shares 651) |
| 552  | Hardware – LED `on()` called before `init()` |
| 553  | Hardware – LED `off()` called before `init()` |
| 554  | Hardware – LED `toggle()` called before `init()` |
| 555  | Hardware – LED `blink()` called before `init()` |
| 432  | EEPROM – `EEPROM.begin()` failed |
| 212  | Adapter/PIR – PIR hardware failed to initialise |
| 222  | Adapter/PIR – failed to re-attach interrupt |
| 231  | Adapter/Serial – mesh transmit unavailable |
| 232  | Adapter/Serial – invalid frame length |
| 233  | Adapter/Serial – frame buffer overflow |
| 234  | Adapter/Serial – message encoding failed |
| 235  | Adapter/Serial – protobuf decode failed |
| 236  | Adapter/Serial – transmit function not set |
| 121  | Adapter base – transmit function not set |
| …    | *(extend as needed)* |

> **Note** Codes marked with * share a number intentionally; they point to the same high-level problem class. Consult the log output for precise context.

## Adding a new code

Always use the helpers in `src/error/Error.h` – never call `ErrorCore` directly.

```cpp
#include "src/error/Error.h"

// Mesh peer-list overflow in Mesh module (sub-code 0x1)
constexpr uint8_t MESH_PEER_OVERFLOW = 0x01;

lattice::err::fail(utils::ErrorType::MEMORY_ERROR,
                      "Peer vector overflow");
// Shows 0x431 (Memory / Mesh / 1) on the display and blinks the
// error LED four times before performing a safe reboot (if configured).
```

Remember to update the table above when introducing a new code.
